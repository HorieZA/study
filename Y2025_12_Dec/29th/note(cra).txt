App.js 참조
01. className을 쓰는 이유는 class가 별도로 예약이 되어있으므로 사용불가
jsx에서는 {}에 변수를 사용할때 ""를 사용하지 않음
02. 부모 태그는 1개만 있어야함, 그렇다보니 다음과 같이 빈태그를 기본적으로 사용 <></>
빈태그 사용 시 디자인 또는 퍼블리싱에 사용되는 태그 적용될 걱정이 없어짐
=> index.js에 있는 render 함수에서도 사용 가능
03. 호출 시 app()을 사용하지 않고 app을 사용하는 이유는 해당 값을 함수형식으로 호출하기 때문.
04. React 사용 시 함수명는 무조건 첫자는 대문자로 시작해야함(대문자 미사용 시 실행이 안됨)
function App() => export default App;
이유 : jsx에서 사용하기 위하여 html를 구분하기 위해 첫자는 대문자로 사용
05. arr을 list로 명칭을 변경하였을때, list이 아닌 arr로 사용 시 "?"를 사용하는 이유는 "undefined"가 발생하여도 오류가 나지 않게 하기 위함
ex) arr?.map((v, i) => <section key={i}>{v}</section>)

index.js 참조
01. const등 선언부를 제외한 입력 함수는 render 함수안에 작성해야 값이 정상적으로 출력됨
02. <React.StrictMode> 함수는 개발 단계에서만 사용 가능
<React.StrictMode><App /></React.StrictMode>
배포시에는 아래와 같이 사용해야함
<><App /></> or <App />
"<React.StrictMode>" 함수를 적용한 상태에서 "console.log();" 사용시 값이 2번씩 발생 할 수 있으므로 배포시 제외해야함.
이유 : <React.StrictMode> 함수는 "그리기위함"과 "감시하기위함" 2가지로 호춯되기 때문에 2번씩 호출됨
