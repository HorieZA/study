Page1.jsx, Page2.jsx 참조
1. useState("") =>()안에는 인자갑을 담는 부분
2. useState()는 읽기 전용이므로 useState()의 값을 가져와서 설정된 값을 수정 할 수 없다.
	const [data, setData] = useState(0)
	[data와, setData의 의미] => ["useState에서 return 하는 값이 담겨있는 읽기전용 변수", "앞의 값을 가져와서 변경하기 위한 함수"]
3. react 사용시 함수가 시작되면 초기화가 기본이기때문에 초기화가 안되게 하기 위해서 useState()를 이용
4. react 사용시 전역변수를 사용하면 안됨
	이유 : react는 내부에서만 활용하는 부분이며, 해당 전역변수가 다른곳에서 사용중인 경우도 존재할 수 있기 때문
	
	
Page3.jsx 참조
1. value={txt} onChange={e => setTxt(e.target.value)} 2개 값이 있어야 값이 출력된다.
2. "onChange" 값을 변경해주는 매개체(?)
3. setTxt()에 e.target.value을 추가했을 경우 submitEvent 함수를 불러오게 되며,
	setTxt()에 고정값을 입력한 경우( ex)setTxt("1") ) 
	빈값에서 1이 추가 되었기 때문에 한번더 submitEvent 함수를 불러고오 이후에는 불러오지 않게 된다.
3. onSubmit => 제출하려고 할때 새로고침(?) 이 발생한다.
	폼이 기본으로 가지고 있는 이벤트를 막아주는 함수(폼이 어디론가 가는것을 막아주는 함수) => e.preventDefault()
4. if (txt === '') return => 예외처리(입력하는 값이 아무것도 없을때 아래에 잆력된 값이 실행되지 않기 위한 것)
5. arr[]이 있고 const b = arr 인 경우 b에 값을 선언해주면 arr도 바뀌게 됨
	이유 : const b = arr은 참조하는 것이며, 위치값은 별로도 설정되어있고 주소값만 변경해주기 때문에 b에 등록한 경우 arr도 변경되게 된다.
7. [a, b] => txt를 넣어주는 위치가 어디에 들어가냐에 따라 최신값이 앞인지 뒤인지 설정할 수 있음
	- setArray([txt, ...array]) =>  해당 방식은 txt에 입력된 값이 최신을 앞에 두도록 설정하겠다는 내용
	- setArray([...array, txt]) =>  해당 방식은 txt에 입력된 값이 최신을 뒤에 두도록 설정하겠다는 내용
	- toReversed().map('') => 최종 보여주는 값을 반전시킨다는 의미
8. setTxt('') => 입력값을 초기화 시켜주겠다는 의미
9. defaultValue와 value의 차이
	- defaultValue: "최초 설정값", 한번 설정되면 변하지 않음 (HTML 기본 동작과 유사).
	- value: "현재 값", 상태와 동기화되어 계속 변함 (React의 제어).


Page4.jsx 참조
1. [...a] => 깊은 복사라고 하며 저장되는 위치가 새로 생성된다고 이해하면 된다.
	ex) const c = [...a]은 const c = [a[0]]과 같다.
	
	
제악을 정상적으로 걸려면 const로 선언해야 한다
var 혹은 let을 사용하면 변수가 바뀌게 됨

다음과 설정이 되어있을때 setA와 if문은 같은 함수 안에서 같이 쓸수가 없다.
이유 : setA인 useState() 함수는 event가 끝나고 나서야 setA가 셋팅되므로, 같이 쓸수가 없다.
같이 둘 경우 비정상 동작임
var[a, setA] = useState("A")
const event = () => {
    setA("A")
    if(a === "AA") return
}
