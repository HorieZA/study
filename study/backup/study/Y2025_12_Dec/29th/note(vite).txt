npm create vite@latest vite

App.jsx 참조
01. vite에서 public폴더는 정적파일을 저장하는곳
02. 이미지는 정적파일이 아니므로 대부분 assets에서 저장하여 사용 (단, 이미지는 백앤드에서 관리)
03. ".css"는 정적파일이 아니고 jsx에서만 사용되므로 브라우저에서는 보이지 않음
04. React 사용 시 html에서는 css를 관리하지않음....
05. React 사용 시 애로우펑션 함수를 많이 사용
06. 지역 관련 함수는 지역에 넣어서 관리하는 것을 추천
07. 지역 관련 함수를 대문자로 사용하여도 되나, 콜백 함수로 사용시에는 굳이 대문자로 사용할 필요가 없음
08. 값이 정상적으로 출력하는지 확인하기 위해서는 "{}"를 이용하여 콘솔로그 사용
09. jsx에서는 문자열을 사용하지 않는것이 좋다
이유 : jsx용 오브젝트가 따로 있기 때문이며, 사용시 
<div dangerouslySetInnerHTML={{ __html: 변환될값 }}></div>을 필수로 입력해야됨
10. DOM 형식으로 만들어야 함
11. 화면출력용을 return 값에 넣어줘야함
12. 하나의 파일에 관리가 가능하지만, 개별로 파일을 생성하여 관리할수 있음.
=> 선택은 자유
ex1) import Main, {Page1, Page2} from './page/Common.jsx'
ex2) import {Page1, Page2} from './page/Common.jsx'
      import Main from './page/Main.jsx'
